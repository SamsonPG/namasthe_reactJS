1. Difference between Named Export, Default Export, and * as Export?
Named Export: Allows you to export multiple functions, objects, or variables from a module by name. You can import them individually using the same names.
Default Export: Allows you to export a single function, object, or variable as the default export of a module. You can import it using any name you choose.
* as Export: This syntax allows you to import all exports from a module and assign them to a single variable.
2. Importance of config.js file
A config.js file is important for centralizing configuration settings and parameters used throughout the application. It helps in easily managing and updating configuration options without needing to modify multiple files.
3. What are React Hooks?
React Hooks are functions that enable functional components to use state and other React features without writing a class. They allow you to use state and other React features in functional components.
4. Why do we need a useState Hook?
The useState Hook in React allows functional components to add state management capabilities. It enables components to keep track of changing data and trigger re-renders when the state updates.
